Управление через командную строку довольно распространено, поскольку позволяет направлять выходные данные из команды в файл с
помощью `>` или использовать `|` для отправки выходных данных через пайп другой команде. Однако иногда нам требуется нечто большее.
Например, вы можете захотеть сохранить выходные данные в файл и просмотреть их в терминале в режиме реального времени.
Чтение файла назначения по мере его записи не позволяет этого сделать, поскольку данные перед записью буферизуются, но есть команда, которая может это сделать,
и она называется *tee*. Название хорошо отражает её назначение – она действует как тройник в трубопроводе, разделяя поток на две части.
Чтобы сделать то, что мы только что описали, вам нужно использовать:

```
$ somecommand | tee somefile
```

*Tee* эффективно создает две копии передаваемых в него данных: одну отправляет в именованный файл, а другую - на стандартный вывод, обычно на терминал.
Обратите внимание, что если файл существует, он перезаписывается без предупреждения. Используйте опцию **-a** (или **--append**), чтобы добавить данные
в существующий файл - *tee* все равно создаст файл, если он не существует, так что вы можете привыкнуть использовать **-a** по умолчанию.

Что, если вы хотите отправить вывод вашей команды в другую программу, такую как *xargs*, и по-прежнему видеть ее результаты? Здесь нет файла.
Ну, есть, потому что в Linux “все есть файл", даже стандартный вывод вашего терминала. Попробуйте что-нибудь вроде этого:

```
$ somecommand | tee /dev/tty | xargs ...
```
